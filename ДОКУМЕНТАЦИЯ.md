### Принцип работы: один CSS/JS — много регионов

Все региональные страницы подключают **одни и те же** файлы:
```html
<link rel="stylesheet" href="../css/style.css">
<script src="../js/script.js"></script>
```

Специфика каждого региона (телефон, цены, тексты, мета-теги) — прямо в HTML-файле региона. Это упрощает поддержку: поправил CSS один раз — обновилось везде.

## 2. Корневой index.html — гео-редирект

**Файл:** `/index.html`

Текущая реализация: **немедленный редирект на Москву** (временное решение).

```html
<meta http-equiv="refresh" content="0;url=./moskva/">
<script>window.location.replace('./moskva/');</script>
```

### Планируемая реализация — автоматическое определение региона по IP

Когда сайт будет на хостинге, нужно заменить на полноценное гео-определение:

```javascript
// Запрашиваем IP-геолокацию
fetch('https://ip-api.com/json/?fields=regionCode,city&lang=ru')
  .then(r => r.json())
  .then(data => {
    const city = (data.city || '').toLowerCase();
    const region = (data.regionCode || '');

    // Логика маршрутизации по городу/региону
    if (city.includes('краснодар') || region === 'KDA') {
      window.location.replace('./krasnodar/');
    } else if (region === 'MOW' || city.includes('москв')) {
      window.location.replace('./moskva/');
    } else if (region === 'MOS') {
      // Московская область (не город Москва)
      window.location.replace('./mo/');
    } else {
      // По умолчанию — Москва (главный регион)
      window.location.replace('./moskva/');
    }
  })
  .catch(() => {
    // Если API не ответил — редирект по умолчанию
    window.location.replace('./moskva/');
  });
```

**Важно:** ip-api.com бесплатен до 45 запросов/минуту. Для высокой нагрузки — заменить на платный сервис (ipinfo.io, DaData) или серверное определение региона через Nginx/PHP.

**Таймаут:** обязательно добавить `setTimeout` на 1.5-2 секунды — редирект по умолчанию на Москву, если API завис.

## 3. Региональные страницы — структура HTML

Каждая страница `/регион/index.html` начинается с конфигурационного блока:

```html
<head>
  <!-- Обязательный блок в самом начале <head> -->
  <script>
    var REGION = 'moskva';   // Идентификатор региона (используется в JS и webhook)
    var WEBHOOK = '';         // URL для отправки заявок (пока пустой — логирует в console)
  </script>
</head>
```

### Переменные региона

| Переменная | Описание | Пример |
|------------|----------|--------|
| `REGION` | Идентификатор региона | `'moskva'`, `'mo'`, `'krasnodar'` |
| `WEBHOOK` | URL куда отправляются заявки (POST JSON) | `'https://hook.make.com/...'` |

## 4. Секции страницы (в порядке сверху вниз)

### 4.1 Топбар (`div.topbar`)
Тонкая синяя полоса над шапкой. Содержит:
- Иконка региона: `📍 Москва`
- Время работы: `Работаем: 8:00 - 21:00, без выходных`
- Email и ссылки на WhatsApp/Telegram (текстовые)

### 4.2 Шапка (`header#site-hdr`)
Залипает при скролле (sticky). Содержит:
- Логотип: `🪟 Окна-Сервис` + подпись региона под логотипом
- Навигация: Услуги / О нас / Видеодиагностика / Отзывы / Контакты
- Телефон (крупный, кликабельный)
- Бургер-меню для мобильных

При скролле добавляется класс `.scr` (усиливает тень).

### 4.3 Героический блок (`section#hero`)
Двухколоночная сетка:
- **Левая:** заголовок H1, подзаголовок, 4 буллета с преимуществами, 2 CTA-кнопки
- **Правая:** карточка со статистикой (10+ лет, 5000+ ремонтов, 2 года гарантии, цена от...)

На мобильных правая колонка скрывается (`display:none`).

Особенность: в фоне работает canvas-анимация частиц (точки + линии между ними). Только на десктопе, запускается в `script.js`.

### 4.4 Преимущества (`section#advantages`)
3 карточки `.adv-card` с иконками. На них работает 3D-тилт при наведении.

### 4.5 Квиз (`section#quiz`)
3-шаговая форма для получения стоимости:
- Шаг 1: выбор проблемы (12 вариантов)
- Шаг 2: тип окна (ПВХ / алюминий / дерево)
- Шаг 3: имя + телефон

При отправке вызывает `submitQ()` → `send()` → POST на `WEBHOOK`.

### 4.6 Услуги и цены (`section#services`)
12 карточек `.svc-card` в сетке 4×3. Каждая карточка — кнопка "Оставить заявку" → открывает модалку `openM('Название услуги')`.

**Цены разные по регионам!** В Краснодаре от 150 ₽, в Москве от 350 ₽, в МО от 350 ₽.

### 4.7 Типы окон (`section#window-types`)
3 карточки по типам (ПВХ / алюминий / дерево) + строка брендов (REHAU, VEKA и др.).

### 4.8 О нас / Почему выбирают (`section#why-us`)
Двухколоночная сетка:
- **Левая:** заголовок + 4 статистики (`.why-stat`) с анимацией появления
- **Правая:** список преимуществ с иконками (`.why-list`)

Статистики анимируются через `IntersectionObserver` — появляются с задержкой 0.1 / 0.25 / 0.4 / 0.55 сек.

### 4.9 Как мы работаем (`section#how-it-works`)
5 шагов на тёмно-синем фоне. Горизонтальная линия между кружками шагов.

### 4.10 Видеодиагностика (`section#video-consult`)
Блок с тёмно-синим фоном. Левая часть — текст + кнопки мессенджеров (WA / TG / Max). Правая — форма записи на видеоконсультацию.

Кнопки мессенджеров в этой секции — стеклянный стиль `.vid-msg-btn` (полупрозрачный белый фон + blur).

### 4.11 Акция (`section#promo`)
Оранжевая карточка с текущей акцией. Содержимое меняется вручную в HTML.

### 4.12 Антифрод (`section#antiscam`)
4 карточки красного цвета — как нечестные компании обманывают клиентов. + зелёный блок CTA.

### 4.13 Отзывы (`section#reviews`)
3 карточки `.rev-card` с текстом, звёздами и именем клиента. На них работает 3D-тилт.

### 4.14 Контакты (`section#contacts`)
Двухколоночная сетка:
- **Левая:** телефон, email, регион, время работы, кнопки мессенджеров
- **Правая:** форма онлайн-заявки

### 4.15 Футер (`footer`)
- Логотип + описание компании + иконки соцсетей/мессенджеров (SVG)
- 2 колонки ссылок: Услуги / Информация
- Нижняя строка: копирайт + телефон

### 4.16 Плавающие кнопки (фиксированные)
Правый нижний угол, всегда видны при скролле:
- WhatsApp (зелёный)
- Telegram (синий)
- Max (сине-фиолетовый градиент)
- Телефон (оранжевый)

На мобильных добавляется `.float-ph-mob` — полоса с телефоном внизу экрана.

### 4.17 Модалка заявки
`.modal-ov` — оверлей, внутри форма с именем и телефоном. Вызывается кнопками услуг `openM('название')`.

### 4.18 Cookie-бар
Появляется снизу при первом заходе. После клика "Принять" прячется и сохраняется в localStorage.

### Анимация появления элементов

**Классы:**
```css
.fu     → начальное состояние: opacity:0, translateY(30px)
.fu.vis → конечное состояние: opacity:1, translateY(0)
```

`.vis` добавляется через `IntersectionObserver` в `script.js` когда элемент входит в поле зрения.

**Задержки** (inline style на элементе):
```html
<div class="fu" style="transition:opacity .7s ease .1s, transform .7s ease .1s">
```
Обязательно писать полный `transition` инлайн (не только `transition-delay`) — иначе браузер может не применить задержку корректно.

```
    ├── Ripple на кнопках
    └── Magnetic CTA buttons
```

### Функция `send(data)`

```javascript
async function send(data) {
  var payload = Object.assign({}, data, {
    region: window.REGION || '',  // из конфига страницы
    utm: getUTM(),                // из localStorage
    url: location.href,
    ts: new Date().toISOString()
  });
  // Если WEBHOOK пустой — только логирует в console
  // Если WEBHOOK задан — POST JSON на указанный URL
}
```

Все формы на сайте вызывают `send()`. Webhook можно подключить к Make (Integromat), n8n, Zapier, Telegram-боту и т.д.

### IntersectionObserver (анимации появления)

```javascript
var obs = new IntersectionObserver(function(entries) {
  entries.forEach(function(e) {
    if (e.isIntersecting) {
      var el = e.target;
      requestAnimationFrame(function() {
        requestAnimationFrame(function() {
          el.classList.add('vis'); // двойной rAF — гарантирует что браузер успел отрисовать opacity:0
        });
      });
      obs.unobserve(el); // наблюдаем каждый элемент только один раз
    }
  });
}, { threshold: 0.1, rootMargin: '0px 0px -40px 0px' });

document.querySelectorAll('.fu').forEach(function(el) { obs.observe(el); });
```

## 7. Региональная система — как добавить новый регион

### Шаги для нового города (например, Санкт-Петербург)

**Шаг 1. Создать папку и скопировать страницу**
```
Скопировать moskva/index.html → spb/index.html
```

**Шаг 2. Поменять конфиг в начале `<head>`**
```html
<script>var REGION='spb'; var WEBHOOK='';</script>
```

**Шаг 3. Обновить мета-теги**
```html
<meta name="description" content="Ремонт окон в Санкт-Петербурге...">
<meta name="geo.region" content="RU-SPE">
<meta name="geo.placename" content="Санкт-Петербург">
<title>Ремонт окон в Санкт-Петербурге — Окна-Сервис</title>
```

**Шаг 4. Заменить все упоминания города в текстах**
- Топбар: `📍 Санкт-Петербург`
- Логотип: подпись под лого `Санкт-Петербург`
- Герой: `h1` — "Ремонт окон в Санкт-Петербурге"
- Секция why-us: "Почему петербуржцы выбирают нас"
- Тексты услуг, отзывов и т.д.

**Шаг 5. Заменить телефон**
Найти и заменить все `+7XXXXXXXXXX` на реальный номер СПб.

**Шаг 6. Скорректировать цены** (если отличаются от Москвы)

**Шаг 7. Добавить геолокацию в корневой `index.html`**
```javascript
} else if (city.includes('санкт') || city.includes('петербург') || region === 'SPE') {
  window.location.replace('./spb/');
```

**Шаг 8. Создать SEO-файл**
```

```

**Шаг 9. Зарегистрировать в Яндекс.Вебмастере**
- Добавить `/spb/` как отдельный раздел
- Привязать регион: Санкт-Петербург

**Шаг 10. Создать карточку Яндекс.Бизнес для СПб**

## 10. Формы и отправка заявок

### Как подключить Webhook

1. Зарегистрироваться на Make.com (или использовать n8n, Zapier)
2. Создать сценарий: **Webhook → Telegram-бот** (или email, CRM)
3. Скопировать URL вебхука в каждую региональную страницу:
```html
<script>var REGION='moskva'; var WEBHOOK='https://hook.eu1.make.com/XXXXXX';</script>
```

```json
{
  "form": "quiz",              // quiz | modal | video | contact

  "problem": "Продувает",      // только для quiz
  "win_type": "Пластиковые",  // только для quiz

  },
  "url": "https://okna-service.ru/moskva/",
  "ts": "2026-05-26T10:30:00.000Z"
}
```

## 12. Яндекс.Метрика

ID счётчика: **92129012**

Код вставлен перед закрывающим `</body>` на каждой региональной странице. При добавлении нового региона — вставить тот же код.

Цели для настройки в Метрике:
- `quiz_submit` — отправка квиза
- `modal_submit` — заявка через модалку (кнопки услуг)
- `video_submit` — запись на видеодиагностику
- `contact_submit` — форма в блоке контактов

## 13. Адаптив (мобильные устройства)

Breakpoints в CSS:
- `≤ 1024px` — сетка услуг 3 колонки, шаги 3 колонки
- `≤ 768px` — всё в 1 колонку, скрывается nav-меню, показывается бургер
- `≤ 480px` — мелкие правки квиза и сетки

WOW-эффекты (курсор, тилт, частицы) — **отключены на touch-устройствах**:
```javascript
if ('ontouchstart' in window) return;
```

На мобильных отображается фиксированная полоса звонка внизу экрана (`.float-ph-mob`).

## 14. Локальный просмотр

Открыть файл `открыть сайт.bat` — он запускает `moskva/index.html` в браузере.

**Ограничение:** при просмотре через `file://` CSS и JS загружаются корректно (относительные пути `../css/` работают). Но некоторые функции (fetch, геолокация) требуют веб-сервера.

Для полноценного локального тестирования — запустить локальный сервер:
```bash
# Python 3
cd "путь/к/папке/Сайт новый"
python -m http.server 8080
# Открыть в браузере: http://localhost:8080
```
